# Пример 1: перенос нетиповых реквизитов на отдельную вкладку

Реальный кейс из практики 1С-разработчика. Исходный промпт был короткой заметкой, улучшенная версия - структурированное ТЗ для исполнителя.

## До

```
Задача - все нетиповые реквизиты в типовых объектах вывести на вкладку
"Доп. реквизиты Айс". Вкладку разместить самой последней. Если добавлен
только один реквизит, Айс_КодИсточника77, то ничего не делаем. Подготовь
список объектов, проанализируй формы элементов, определи, куда и как
добавить вкладку.
```

## После

```
Проанализируй все типовые объекты конфигурации и подготовь список тех,
в которых присутствуют нетиповые реквизиты. Изучи формы элементов этих
объектов. Добавь в каждую форму новую вкладку "Доп. реквизиты Айс"
последней по порядку. Перенеси на вкладку все нетиповые реквизиты,
кроме случая, когда единственный нетиповой реквизит это
Айс_КодИсточника77 (тогда изменения не вносятся). Для каждого объекта
укажи:

1. Наименование объекта.
2. Список нетиповых реквизитов.
3. Название используемой формы элемента и текущую структуру вкладок.
4. Рекомендуемые изменения по размещению вкладки и переносу реквизитов
   с учетом логики интерфейса.

Сформируй вывод в виде удобной таблицы или структурированного списка
для дальнейшего внедрения.
```

## Что изменилось

1. **Императивная форма.** "нужно вывести" и "необходимо перенести" заменены на активные "Выведи", "Перенеси".
2. **Цель в первом предложении.** "Проанализируй все типовые объекты... подготовь список" задает цель сразу, без преамбулы.
3. **Шаги анализа пронумерованы по смыслу.** Сначала идет анализ (типовые объекты, формы элементов), потом формирование результата.
4. **Граничный случай вынесен отдельно.** Вместо скобочной вставки "если добавлен только один реквизит..." получилось полноценное предложение с конструкцией "кроме случая, когда ...".
5. **Термины сохранены буквально.** Айс_КодИсточника77, "Доп. реквизиты Айс", "нетиповые реквизиты" переносятся без изменений.
6. **Формат ожидаемого вывода задан явно.** "удобная таблица или структурированный список" и пронумерованный перечень полей для каждого объекта.
7. **Нет новых требований.** Не появилось упоминаний про тесты, документацию, откат, права доступа - всего того, чего не было в исходном тексте.
