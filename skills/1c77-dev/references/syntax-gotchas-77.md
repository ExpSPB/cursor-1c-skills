## 1. Логический тип в условиях

В 7.7 в `Если <выр> Тогда` / `ИначеЕсли <выр> Тогда` / `Пока <выр> Цикл` выражение должно быть
**результатом операции сравнения или логической операции**. Голый вызов функции или число логическим
выражением не считается, даже если функция возвращает 0/1.

```
// ОШИБКА компиляции: "Выражение должно иметь логический тип"
Если ПустаяСтрока(текОКПО) Тогда ... КонецЕсли;
// после НЕ: "Ожидается логическое выражение"
ИначеЕсли НЕ ПустаяСтрока(текКПП) Тогда ...

// ПРАВИЛЬНО - явное сравнение
Если ПустаяСтрока(текОКПО) = 1 Тогда ... КонецЕсли;
ИначеЕсли ПустаяСтрока(текКПП) = 0 Тогда ...
```

Это касается **любой** функции в условии, не только `ПустаяСтрока`. Если функция возвращает число-флаг -
сравнивай его явно (`= 1`, `<> 0`, `> 0`). Тот же принцип в тернарном операторе (см. п.5):
условие внутри `?(...)` тоже должно быть сравнением: `?(ПустоеЗначение(Х) = 1, "", СокрЛП(Х))`.

Почему так: в 7.7 нет автоприведения числа к логическому. Компилятор разбирает условие как логическое
выражение и требует операцию отношения. Привычка из 8.x (`Если Функция() Тогда`) даёт ошибку.

## 2. Нет типа Булево

В 7.7 нет литералов `Истина`/`Ложь` как типа Булево (в большинстве релизов). Логика выражается числами:
`0` - ложь, не ноль - истина, но **только** как результат сравнения/логической операции в условии.
Флаги в реквизитах хранятся как `Число(1.0)` со значениями 0/1.

- Присваивание флага: `тз.Флаг = ?(спр.Флаг = 1, 1, 0);` (нормализация к 0/1).
- Проверка флага: `Если спр.Флаг = 1 Тогда`, а не `Если спр.Флаг Тогда`.

## 3. Функции, которых нет или они другие

- `ПустаяСтрока(Стр)` - в типовых 7.7 обычно определена в глобальном модуле и возвращает **число** 0/1
  (не Булево). В условии требует сравнения (см. п.1). Если в конкретной базе её нет - проверяй пустоту
  через `СокрЛП(Стр) = ""`.
- `ЗначениеЗаполнено(...)` - функции 8.x в 7.7 нет. Аналоги: `ПустоеЗначение(Знач) = 1` (для значений
  ссылочного/любого типа, глобальная), `СокрЛП(Стр) = ""` (для строк), `Объект.Выбран() = 1` (для ссылок).
- Тернарный оператор `?(Условие, ЗначениеЕслиИстина, ЗначениеЕслиЛожь)` - есть, но `Условие` обязано быть
  сравнением.
- Нет `Новый` для большинства объектов - вместо `Новый` используется `СоздатьОбъект("Тип")` (см. п.6).
- Работа с файлами: нет объектов 8.x. Текст пишут через `ТекстовыйДокумент`/`СоздатьОбъект("Текст")`
  или внешнюю компоненту `FileSystemObject` (Scripting) для скорости на больших объёмах.

Если не уверен, есть ли функция в конкретной базе - проверь глобальный модуль через MCP `1c77-metadata`
(`get_global_module` / `search_in_modules`), не полагайся на знание 8.x.

## 4. Безопасный доступ к реквизитам

Реквизит может отсутствовать в конкретной редакции конфигурации (особенно при доработках под клиента).
Обращение к несуществующему реквизиту - ошибка времени выполнения. Чтобы выгрузка не падала на «чужой»
базе, оборачивай чтение необязательных реквизитов в `Попытка/Исключение`:

```
тзСтрока.Поле = "";
Попытка
    тзСтрока.Поле = СокрЛП(спр.Поле);
Исключение
КонецПопытки;
```

Инициализируй переменную значением по умолчанию ДО `Попытка`, чтобы при отсутствии реквизита осталось
корректное пустое значение.

## 5. Идиомы выгрузки значений

Проверенные шаблоны присваивания (из реального патчинга обработки выгрузки). Тип колонки ТЗ создаётся
через `тз.НоваяКолонка("Имя","Тип"[,длина[,точность]])`, заполнение - в цикле обхода справочника:

| Вид данных в 7.7 | Колонка ТЗ | Присваивание (внутри Попытка/Исключение) |
|------------------|-----------|------------------------------------------|
| Строка | `НоваяКолонка("Х","Строка",длина)` | `тз.Х = СокрЛП(спр.Х);` |
| Число | `НоваяКолонка("Х","Число",прец,точн)` | `тз.Х = спр.Х;` |
| Дата | `НоваяКолонка("Х","Дата")` | `тз.Х = спр.Х;` |
| Флаг (Число 1.0) | `НоваяКолонка("Х","Число",1)` | `тз.Х = ?(спр.Х = 1, 1, 0);` |
| Ссылка -> код | `НоваяКолонка("Х","Строка",12)` | `тз.Х = ?(ПустоеЗначение(спр.Х) = 1, "", СокрЛП(спр.Х.Код));` |
| Перечисление -> имя | `НоваяКолонка("Х","Строка",50)` | `тз.Х = СокрЛП(спр.Х);` (имя значения) |
| Ссылка/строка (смешанно) | `НоваяКолонка("Х","Строка",100)` | `тз.Х = СокрЛП("" + спр.Х);` |

Генерация таких блоков для десятков полей автоматизируется - см. `scripts/patch_by_marker.py`
(вставка по якорю) и образец генераторов в `Documents/patch_ert_module.py` проекта.

## 6. Работа с объектами через СоздатьОбъект

```
спр = СоздатьОбъект("Справочник.Номенклатура");
спр.ВыбратьЭлементы();
Пока спр.ПолучитьЭлемент() = 1 Цикл      // = 1, не голый вызов
    Если спр.ЭтоГруппа() = 1 Тогда Продолжить; КонецЕсли;
    // ... работа с спр.Реквизит
КонецЦикла;

тз = СоздатьОбъект("ТаблицаЗначений");
тз.НоваяКолонка("Код","Строка",9);
```

Запрос - через `СоздатьОбъект("Запрос")` и текст на языке запросов 7.7 (отличается от 8.x; проверяй
через `validate_query` MCP `1c77-metadata`).

## 7. Строки, конкатенация, пустые значения

- Конкатенация - оператор `+`. Приведение значения к строке: `"" + Значение`.
- Обрезка пробелов: `СокрЛП` (с обеих сторон), `СокрЛ`, `СокрП`.
- Пустая ссылка/значение: `ПустоеЗначение(Знач)` возвращает число (`= 1` если пусто).
- Подстроки: `Лев(Стр, N)`, `Прав(Стр, N)`, `Сред(Стр, Поз, Длина)`, поиск `Найти(Стр, Подстр)`.
- Длина строки: `СтрДлина(Стр)`.

## 8. Чек-лист перед сборкой

Пройди по списку перед `build_ert.py` (быстрее, чем ловить на компиляции в 7.7):

- [ ] Все `Если`/`ИначеЕсли`/`Пока` содержат сравнение, а не голый вызов функции/число.
- [ ] `НЕ Функция(...)` заменено на `Функция(...) = 0` (или обёрнуто в скобки с явным сравнением).
- [ ] Тернарники `?(...)` имеют сравнение в условии.
- [ ] Необязательные реквизиты читаются через `Попытка/Исключение` с инициализацией по умолчанию.
- [ ] Нет функций/конструкций 8.x (`Новый`, `ЗначениеЗаполнено`, объекты файлов 8.x).
- [ ] Запросы проверены `validate_query` (MCP `1c77-metadata`).
- [ ] Файл сохранён в CP1251, переводы строк не побиты (CRLF).
